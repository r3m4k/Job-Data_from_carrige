Программа для стягивания данных 

Используемые параметры командной строки:
	"profile" 
	---> Возможные варианты: "v", "vertical", "h", "horizontal"
	---> Позиционный параметр, указывающий на анализируемую величину - вертикальный профиль или горизонтальный план
	---> Пример использования:
		>>> .\tightening.exe v
		>>> .\tightening.exe "v"
		>>> .\tightening.exe "vertical"
		>>> .\tightening.exe h

	"-eh" или "--end_height"
	---> Значение по умолчанию - 0.0
	---> Ожидаемый тип данных - float
	---> Перепад высот между первой и последней точками, указанный в метрах. Используется при анализе вертикального профиля.
	---> Пример использования:
		>>> .\tightening.exe v -eh 1.2

	"-tm" или "--test_mode"
	---> Значение по умолчанию - False
	---> Возможные варианты: True, False
	---> Активация тестировочного режима, при котором будут созданы дополнительные графики, поясняющие шаги выполнения программы. 
	     РЕКОМЕНДАЦИЯ: перед использованием программы после применения тестировочного режима убедитесь, что в папке сохранения не остались дополнительные графики во избежание путаницы.
	---> Пример использования:
		>>> .\tightening.exe v -eh 1.2 -tm 1
		>>> .\tightening.exe "horizontal" -tm 1

	"-f" или "--filter_type"
	---> Возможные варианты: "average_filter", "median_filter"
	---> Значение по умолчанию "average_filter"
	---> Тип используемого фильтра при стягивании данных - - медианный или усредняющий
	---> Пример использования:
		>>> .\tightening.exe v -eh 1.2 -f "median_filter"

	"--gyroscope_rotation"
	---> Значение по умолчанию - False
	---> Возможные варианты: True, False
	---> В тележке, используемой во время написания данной программы, гироскоп расположен перпендикулярно направлению пути, поэтому вертикальный профиль строится на основе крена, а не тангажа. Но если гироскоп будет расположен вдоль пути движения, то вертикальный профиль должен основываться на тангаже, что и учитывается при включении данного флага
	---> Пример использования:
		>>> .\tightening.exe "vertical" -eh 1.2 -tm 1 --gyroscope_rotation 1
